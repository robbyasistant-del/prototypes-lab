Pulse Keeper — Product Report

Date: 2026-02-28 · Genre: Skill timing / rhythm survival · Build: Web MVP polished loop

1) Strategy

Designed as a short-session, high-repeat skill game with daily seeded cadence. Core objective is consistent D1 replay by creating “almost perfect” runs and social brag moments (score + combo). Monetization-fit: rewarded continues, cosmetic beat trails, daily pass missions.

2) SEO / ASO / SEM

  • SEO landing: keywords: “rhythm reaction game”, “timing challenge”, “daily reflex game”. Long-tail article: “How to improve rhythm timing in 5 minutes/day”.
  • ASO: title variants: Pulse Keeper: Rhythm Survival / Beat Zone Reflex. Icon: cyan pulse crossing neon gate.
  • SEM: test TikTok + UAC creatives featuring perfect hit streaks and near-fail clutch clips. CPI goal tier-1: < €0.85.

3) Core Loop

  1. Start daily run (90s).
  2. Hit pulses in timing window.
  3. Build combo → gain score and stability.
  4. Misses break combo + damage stability.
  5. Finish timer (or collapse), compare leaderboard/self-best.

4) Rewards & Retention

  • Session rewards: score medals (Bronze/Silver/Gold/Perfect).
  • Meta rewards: streak chest each 3-day completion.
  • Social rewards: auto-generated share card with combo peak and accuracy.
  • Retention hooks: rotating weekly modifiers (faster lane, fake pulses, mirror mode).

5) KPI Targets

  • D1 retention: 35%+
  • Average sessions/day active user: 2.3+
  • Run completion rate: 72%+
  • Median score spread (p50 to p75): enough skill expression (1.5x).
  • Ad watch rate for continue: > 22% of failed runs.

6) Experiments (Next 2 weeks)

  • A/B perfect-window size (8px vs 10px) to balance frustration.
  • A/B session length (90s vs 60s) for better completion + replay.
  • Ghost replay overlay vs static leaderboard for competitiveness lift.

7) Risks

  • Ceiling too high for casual players → early churn.
  • Perceived repetition without enough daily modifier variety.
  • Accessibility: color-only feedback may reduce usability for some users.
